      "maxConcurrency": "16-32",              # Max concurrent requests per connection
      "cMaxReuseTimes": "64-128",             # Max connection reuse times
      "hMaxRequestTimes": "600-900",          # Max HTTP requests per connection
      "hMaxReusableSecs": "1800-3000",        # Max connection lifetime (seconds)
      "hKeepAlivePeriod": 0                   # Keep-alive period (0=auto: 45s H2, 10s H3)
    }
  }
}

VLESS XHTTP Format (URL-encoded extra with complete parameters):
- extra parameter contains URL-encoded JSON with comprehensive XHTTP settings
- mode=auto for automatic XHTTP mode selection (packet-up/stream-up/stream-one)
- xPaddingBytes: "100-1000" Random padding range for anti-fingerprinting
- scMaxEachPostBytes: "500000-1000000" Max bytes per POST request (packet-up)
- scMinPostsIntervalMs: "10-50" Min interval between POST requests
- noGRPCHeader: false Enable gRPC Content-Type spoofing for CDN compatibility
- noSSEHeader: false Enable Server-Sent Events headers for streaming
- scStreamUpServerSecs: "20-80" Keep-alive interval to prevent CDN timeout
- xmux: Complete multiplexing control for H2/H3 performance optimization

Complete parameter documentation: See XHTTP_DOCUMENTATION_COMPLETE.md

# ============================================================================
# 5. KEY DIFFERENCES BETWEEN PROTOCOLS
# ============================================================================

VMESS:
- Format: base64 encoded JSON
- Has aid (Alter ID) field
- Uses "net" for transport type
- Path format: "/vmess", "/chat", etc.

VLESS: 
- Format: URL-like string
- No aid field (simpler)
- Uses "type" for transport
- More efficient than VMess
- Supports REALITY protocol

TROJAN:
- Format: URL-like string  
- Uses password instead of UUID
- Path must be URL encoded (%2F = /)
- Can bypass some firewalls better

XHTTP:
- Latest transport protocol from Xray project
- Based on official GitHub discussion #4113 "XHTTP: Beyond REALITY"
- HTTP/2 and HTTP/3 support with QUIC compatibility
- Advanced features: Header padding, XMUX multiplexing, uplink/downlink separation
- Better performance on mobile networks and CDN traversal
- Comprehensive extra raw JSON parameters for fine-tuning
- Anti-censorship via traffic obfuscation and pattern randomization
- Supports packet-up (compatibility), stream-up (performance), stream-one (efficiency) modes
- See XHTTP_DOCUMENTATION_COMPLETE.md for detailed parameter explanations

REALITY:
- Newest security protocol
- No TLS certificates needed
- Mimics real websites (www.microsoft.com)
- Very hard to detect/block
- Direct connection (bypasses Nginx)
